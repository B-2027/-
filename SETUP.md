# 🛠️ دليل الإعداد والتطوير

## المتطلبات المسبقة

- Git مثبت على جهازك
- محرر نصوص (VS Code, Sublime, إلخ)
- حساب على GitHub

---

## ⚙️ خطوات الإعداد

### 1. استنسخ المستودع

```bash
git clone https://github.com/B-2027/-.git
cd -
```

### 2. أنشئ فرع جديد

```bash
git checkout -b feature/اسم-الميزة
```

### 3. عدّل الملفات

اختر الملف الذي تريد تعديله من مجلد `categories/`

### 4. أضف التغييرات

```bash
git add .
```

### 5. اكتب رسالة Commit واضحة

```bash
git commit -m "إضافة مقولات جديدة في فئة [الاسم]"
```

### 6. ارفع التغييرات

```bash
git push origin feature/اسم-الميزة
```

---

## 📝 معايير كتابة المقولات

### الصيغة الصحيحة

```markdown
- "النص الكامل للمقولة بالضبط" - المصدر (اختياري)
```

### أمثلة صحيحة

✅ صحيح:
```markdown
- "النجاح يبدأ برغبة قوية" - حكمة عربية
- "العلم نور" - من الحديث الشريف
- "لا شيء مستحيل" - نابليون بونابرت
```

❌ خطأ:
```markdown
- النجاح يبدأ برغبة قوية
- "النجاح" - بدون نص كامل
- "النجاح يبدأ برغبة قوية" - بدون مصدر (مقبول لكن أفضل مع مصدر)
```

## 🔍 قائمة التحقق

قبل إرسال Pull Request:

- [ ] المقولات موثوقة ودقيقة
- [ ] لا توجد أخطاء إملائية
- [ ] الصيغة متسقة مع باقي الملف
- [ ] المصادر صحيحة
- [ ] لا توجد نسخ مكررة
- [ ] الملف تم حفظه بصيغة UTF-8

---

## 🚀 التطوير المتقدم

### إضافة فئة جديدة

1. أنشئ ملف جديد في `categories/` باسم الفئة
2. اتبع الصيغة الموجودة في الملفات الأخرى
3. أضف الرابط في `README.md`

### مثال:

```markdown
# 🎨 الإبداع والفن

## الإلهام الإبداعي
- "الإبداع يأتي من الجرأة على التجربة"
- "الفن يعبر عما لا تستطيع الكلمات تعبيره"

...
```
